Writing Java Code

Java Source Files

  • In Java, almost everything belongs inside of a class, i.e., inside…
    •                 
                class Example{
                     ...
                }

  • There should generally be only one class per source file, and the names should match… for example a class named Student should be placed in Student.java, and be the only class in the Student.java file.

  • JAVA IS CASE SENSITIVE
    – The sooner you accept this reality, the better.
    – There are a few things that can be expressed both in upper and lower case, but not many.

Java Language Elements

  • Java source files contains various elements, such as:
    • Classes
    • Variables
    • Methods
    • Comments
    • Reserved words
    • Modifiers
    • Statements
    • Block

Classes

  • A class consists of two fundamental kinds of things:
    • Variables
    • Methods
  • A class describes its objects–The data (variables) that each object of the class will have–The operations (methods) defined on those objects
  • A class is a way to model an OO concept
  • We will revisit this later. First, we have to learn more about the language

Packages

  • Java classes are organized into packages
    • Packages are directories on the file system
  • A class without a package statement is in the default package
    • A package with no name
  • Class names must be unique within a package
  • The Java Core API library is divided up into packages
    • java.lang, java.net, java.io, etc

Identifiers

  • Names used for classes, variables and methods
  • Can be made up of any length of:
    • Letters
    • Digits
    • Underscore (_)
    • Dollar ($)
    • Cannot begin with a digit
    • Cannot be a Java keyword (we'll see these soon)
    • Packages are directories on the file system
  • Java is case sensitive
    • User and user are completely different identifiers

Take a look at the list of correct/incorrect identifiers below

                    MyClass
                    myVariable
                    MY_VARIABLE
                    x
                    _myvariable
                    $myvariable
  • Invalid identifiers:
    •                      
                     My Variable      // contains a space
                     9pins begins     // with a digit
                     a+c+             // + is not an alphanumeric character
                     testing1-2-3 a   // - is not an alphanumeric character

Java Naming Conventions

  • While not strictly required (by the compiler), pretty much law
  • Class names begin with a capital letter
    • Class examples
      • String
      • System
      • Person
      • HelloWorld
  • Variable and method names begin with a lowercase letter
    • Variable examples:
      • name
      • age
      • initialCount
      • buffer_Size

    • Method examples:
      • getName()
      • setAge()
      • println()
      • calculateInterest()
  • In both cases, subsequent "words" in the name are capitalized

Comments

  • Single line
    • Begin with //
    • Continues to end of line
      •                         
                        // single-line comments
  • Multi-line or "block"
    • Begin with /*and end with */
    • Cannot nest these type of comments
                            
                        /*
                        this comment spans
                        several lines
                        */
  • Javadoc documentation comments
    • Begin with /**and end with */
    • Read by javadoc tool to generate HTML-based API doc (later)

Whitespaces

  • Spaces, blank lines and tabs
    • Used to separate words and symbols in a program
  • Extra white space is ignored by the compiler

Binary Literals

  • In Java 7, the primitives byte, int, short and long can also be expressed using the binary number system
  • In order to specify a binary literal, you add the prefix 0b or 0B to the number
  • The following are examples of binary literals:
    •         
        byte aByte= (byte)0b00100001;     //An 8 –bit ‘byte’ value
        short ahort= (short)0b1010000101000101;       //A 16-bit ‘short’ value
        int integer1 = 0b10100001010001011010000101000101;
        int integer2 = 0b101;     //Some 32-bit ‘int’ values

Underscores in Numeric Literals

  • In Java 7, underscore characters can appear anywhere between digits in a numerical literal
  • This feature allows us to separate groups of digits in numeric literals which can improve the readability of your code
  • The following examples demonstrate valid underscore placement:
    •         
        int binary = 0b1001_1001;    //binary
        int x = 99_9; 
        int y = 99______9;
        float f = 99.9__9F;
        double d = 0_99.99;

Primitive Type Conversion and Casting

  • "Upcasts" to a larger type are implicit and automatic
  • "Upcasts" to a larger type are implicit and automatic
    • Because you might lose data and/or precision
    •          
          float length = 2.1F;                  // 2.1F is shorthand for (float) 2.13
          float width = 4.51F;
          double area = length * width;        // auto upcast to double

          float approxArea = (float) area;    // explicit downcast to float
  • "Upcasts" or conversion may occur during operations
    •                     
                    int + long --> long             // int converted to long
                    float + double --> double      //  float converted to double
                    int + float --> float         //   int converted to float

Methods

Introduction to Methods

  • Remember: Objects are about responsibility
    • Responsibilities are fulfilled by the object's operations that it can perform

    Methods break up the object's functionality into distinct, cohesive units

    Cohesiveness(" being about one thing") ensure the method has a tightly defined job, and not a "do it all here" structure

    Methods should be modular in design, which allows for more code reusability

Defining Methods

    Anatomy of a method:

  • return-type method-name(param-type param-name, ...) { // method "body" -implementation code goes here} 
    •         
        void getName(String name){
          // code...
        }
  • Return type- The data type of the value that a method sent back
    • keyword "void" is used if it returns nothing
    • inside the method the "return" keyword is use to return a value
  • Method name - a valid Java identifier( i.e methodName)
  • Parameters –Each one just like a variable declaration
    • Enclosed in parentheses, separated by commas
    • There can be 0 - infinity parameters
    • () - empty parenthesis use for no parameters
    • Code block - everything the curly braces enclose {}

main Method - Revisited

  • main is a somewhat special method. It is the point of entry for every Java Application
    • The JVM starts your program by invoking the main method 
      •           
            public static void main(String[] args){
              //...
            }
  • When you write a method, you get to pick its signature
    • The calling code must call it by it's proper signature (which you document)
  • You can't pick the signature of main, because we have an agreement with the JVM (which is outside our control)
  • It will call public static void main(String[])
  • You must have this method exactly as the JVM is expecting it, or it can't start your program
